	Operation
		Go
			
				1-go receive from UI Request 
				2-go send Request to c++
				3-go wait for "done" from C++
				4-go read response from C++
				5-go send response to UI
		C++
			
				1-C++ wait for "check" to check Request
				2-C++ Receive Request
				3-C++ Process Request
				4-C++ tell Go that Response is Ready
				5-C++ Write Response
	Definition
		Make Communication Between 2 Languages or Process and Pragrams  on same machine
		Exchange Information, Data and Commands
		Low latency, Low CPU load, high efficiency
	Deep Dive
		Pipe(File)
			First Byte
				Locked for check if Can use the system or no, controlled by host.
			Socket
				Socket is 1024 Byte, Allocated for each Pear
				Part 1(1 - 16 byte)
					Byte 1 to 16 is for This Socket ID
				Part 2(17 - 64 byte)
					Byte 17 
						For State (Active or Inactive)
					Byte 18
						For Flags (Ready,Read ,Seen, Done)
					Byte 19
						For Errors (True and False)
					Byte 20 - 36
						Set Peer Socket ID (Read by host Socket)
					Byte 37 - 62
						Connection Reason (Under 25 char string)
					Byte 63
						Use Pocket, Rooms or Outside file for reading 0->Pocket; 1->Room; 2->File;
							If Rooms or Outside File Will use, Room ID/File path will set on Host Pocket  At first.
					Byte 64
						For Accept or Reject Connection
				Part 3 (65 - 1024 Byte)
					 Socket Pocket
				Pocket 
					480 Byte to each Peer for Small Data Transfering & Connection Setting
					Can use for multi-Reading in Large Data Transfering. Divide Data for Parts and Set Data in Room one by one while stay Connection between Peers on Pocket.
					If Multi-Transfering On, Host Pocket Decide when to start, stream and stop
					First byte in Pocket tell peer if this line read or no. (Reading Flag)
						Byte is 0 (Nothing to Read)
						Host put data
						Host flip byte make it 1(New Data)
						Peer flip byte to 2 (Read and ready to New)
						Host flip byte to 0 (Nothing to Read)
			Room
				Part 1 (1 - 16 Byte)
					Room ID
				Part 2 (17 - 32 byte)
					Room Size
				Part 3 (Byte 33)
					Reading Flag
				Part 4 (Byte 34 to Rest of Room... )
					Store Data
				Place to Store Data(outside the Socket)
				Size is Flexible ,But with fixed Number(1kb, 4kb, 2mb, 16mb... and so on)
				Minimal Size for each Room is 1024 byte (1 Kilo Byte)
		Mapping File
			Points Sockets/Rooms IDs to it position
			On init, Entire File Read and Saved in Ram
			Uses "Scanner" Tool in its Architecture (Simple)
			Depends on fstream, string based.
			If add, file append.
			If Read, Read full Line.
	Architecture
		File Structure
			Single file for both, open file as Reading/Writing.
			Working On low-level IO, Using binary for streaming.
			Contain "Flags" to manage Connection
			Flags are 3, and they Handle and Route Connection
			Contain "Rooms" to store Data
			Contain Mapping System
		Flags
			Ready
				 Peer Can Handshake if this Ready
				Ready for Receiving
			Wait
				During Process Mode
				Peer should wait for Response
			Done
				Peer Can Read Response/Data
				Cannot do Handshake until Read Response
		Rooms
			Room Can be Reached by Name
			Room Can be Added,Deleted and Sized By Main Language (C++)
			Room Store data for a while
			Room Can be free from both Peers
			Room is a Container for Data Exchangement
			There are a Map for Rooms
		Map
			Map is a separated file 
			Map Points to Rooms
			Room Name > Room border
			Map is a separated tool can be used outside
